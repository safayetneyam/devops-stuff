# Services
Types of Services:
- ClusterIP (for internal access)
- NodePort (to access the application on a particular pod)
- Load Balancer (to access the application on a domain name or IP address without using the port number)
- External (to use an external DNS for routing)

## Service: ClusterIP

### ClusterIP - Imperative Command
To expose a pod or deployment:
```yaml
kubectl expose pod <name-of-pod> --port=<port-number>
kubectl expose pod nginx-pod --port=80

kubectl expose pod nginx-pod --name=cluster-service --type=ClusterIP --port=80 --target-port=80

kubectl expose deployment nginx-deployment --name=cluster-service --type=ClusterIP --port=80 --target-port=80
```
- ```--port=80```: Service Port
- ```--target-port=80```: Container Application Port
- ClusterIP is the default Service type, so ```--type=ClusterIP``` is optional.

### ClusterIP - Declarative Command 
```yaml
apiVersion: v1
kind: Service
metadata:
  name: cluster-service
  namespace: default    # optional
  labels:
    env: demo
spec:
  type: ClusterIP       # optional 
  selector:
    env: demo
  ports:
    - name: http
      protocol: TCP
      port: 80
      targetPort: 8080 
```
- It matches the service selector label with the deployment label (pods created by deployment) and exposes it on an internal IP address.
- If ```targetPort: 8080``` is configured, the application inside the selected Pods must actually listen on port 8080. Declaring ```containerPort: 8080``` in the Deployment is optional, but recommended for clarity.

**Important exception**: when ```targetPort``` uses a name, that name must match a named container port:

```yaml
# Service
targetPort: http
```
```yaml
# Deployment
ports:
  - name: http
    containerPort: 8080
```

### Template Generation of ClusterIP
```yaml
kubectl create service clusterip nginx-service --tcp=80:8080 --dry-run=client -o yaml > service.yaml
```

```text
    80   = Service port
    8080 = targetPort on the Pods
```

## Service: NodePort

**Concepts** 
- Request Execution by ```<NodeIP>:<NodePort>```
- NodePort Range: 30000-32767

In the NodePort, the operator deals with three types of ports:
- **NodePort**: A port opened on each Kubernetes node through which the Service can be accessed from outside the cluster.
- **Internal Service Port**: The port on which the Service is available inside the cluster.
- **TargetPort**: The port on the selected Pods where the application is listening.


### NodePort - Imperative Command
```yaml
kubectl create service nodeport nodeport-service --tcp=80:80 --node-port=30001 # --dry-run=client -o yaml
kubectl set selector service/nodeport-service env=demo  # to change the selector
```
the Service will route traffic to Pods containing env:demo in labels.

### NodePort - Declarative Command
```yaml
apiVersion: v1
kind: Service
metadata:
  name: nodeport-service
  labels:
    app: nodeport-service
spec:
  type: NodePort
  selector:
    env: demo
  ports:
    - name: http
      protocol: TCP
      port: 80
      targetPort: 80
      nodePort: 30001
```
