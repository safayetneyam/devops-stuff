# Namespaces
Namespaces are Kubernetes objects that provide isolation of resources within the cluster. In Kubernetes, there are two types of resources:
- **Namespace-scoped Object**: Pods, Deployments, Services, etc.
- **Cluster-scoped Object**: StorageClasses, Nodes, PersistentVolumes, etc.

To check what resources are namespace scoped and what are not:
```yaml
kubectl api-resources --namespaced=true
kubectl api-resources --namespaced=false
```

When a Kubernetes cluster is created, these four namespaces usually exist:
- **default**: Resources are created here when no namespace is specified.
- **kube-node-lease**: Stores node Lease objects used for kubelet heartbeats and failure detection.
- **kube-public**: Contains resources intended to be publicly readable, depending on cluster permissions.
- **kube-system**: Stores Kubernetes system components including the control plane components and cluster-managed resources.


## Namespace Management

### Namespace - Imperative Command

```yaml
kubectl create namespace <name-of-namespace>

kubectl create ns development
kubectl create namespace development
```

### Namespace - Declarative Command

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: development
```

## Namespace Usage

### Imperative Commands
```bash
kubectl get pods -n deployment
kubectl get deploy --namespace deployment

kubectl apply -f nginx-deploy.yaml --namespace demo
kubectl create deploy nginx-demo --image=nginx -n demo
```

### Declarative 
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  namespace: development
spec:
  replicas: 2
  # ...
```

## Notes
- Services in the same namespace can usually be accessed using only the Service name: 
    - ```<service-name>```
    - Example: ```nginx-service```
- Services in a different namespace should be accessed using: 
    - ```<service-name>.<namespace-name>```
    - ```<service-name>.<namespace-name>.svc.cluster.local```
        
        the full DNS is referred to as **FQDN** (Fully Qualified Domain Name)
    - Example: ```nginx-service.production.svc.cluster.local```
- Different namespaces are not automatically blocked from communicating; access may be restricted only by NetworkPolicies or other security rules.
